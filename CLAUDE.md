# Project Instructions

## Communication Style
- Be concise. No flattery, no filler, no "Great question!"
- Answer what was asked. Don't add unrequested features.
- When I ask you to explain, explain clearly but briefly.
- If something is uncertain, say so.

## Working Rules
- Never modify files without being asked.
- Show me what you plan to change BEFORE changing it.
- One task at a time. Don't refactor while fixing a bug.
- When writing code, add a brief comment explaining WHY, not WHAT.
- Use SwiftUI, not UIKit, unless I specify otherwise.
- Target iOS 17+.

## Git Discipline
- Don't commit on my behalf unless I ask.
- Suggest commit messages when work is done.

## Token Conservation
- Don't repeat code I've already seen.
- Don't restate the question back to me.
- Use diffs when showing changes to existing files.
